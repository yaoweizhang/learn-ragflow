# s10 GraphRAG — Minimal Entity-Relation Graph for "How is X related to Y?" Queries

Vector search from s06–s09 finds "documents that mention X" and "documents that mention Y" separately, but never tells you there is an edge X → Y in the corpus. This chapter closes that gap: an LLM extracts `(head, relation, tail)` triples from each chunk, `build_graph` merges them into an in-memory `dict[head] → set[(rel, tail)]`, and `query_graph` returns all 1-hop neighbors of a given entity. The MVP keeps the graph in process memory and answers 1-hop only — RAGFlow's full GraphRAG adds Leiden community detection, community summaries, entity resolution (alias merging), and persistence into the inverted index alongside text chunks; details in [docs/reference/ragflow-notes/graph_extraction.md](../docs/reference/ragflow-notes/graph_extraction.md). The chapter README walks through the demo output, three real-world failure modes (entity ambiguity / aliasing, graph explosion, multi-hop query cost), and how RAGFlow addresses each.
