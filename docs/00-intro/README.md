# `docs/` 目录说明

`docs/` 是**项目级（project-wide）补充阅读**层，与 `s01_..s12_` 是平行的两条不同性质的轨道：

| | `s01_what_is_rag/` 等 | `docs/00-intro/` |
|---|---|---|
| **性质** | 教学单元（每个章节一个） | 概念综述（项目级） |
| **角色** | 一个 30-80 行的 MVP + 跑通指南 | 解释 RAG 是什么、为什么、和什么权衡 |
| **能不能跑** | 能——`python sXX/code.py` | 不能——纯文本 |
| **例子** | `chunk_by_paragraph(...)` 真的去切 chunk | 讲"为什么我们要切" |

## 边界

- **不要把 sXX 的内容搬到 docs/**。教学单元的可运行示例是它的核心价值，docs 只放解释。
- **不要把 docs 的内容搬到 sXX**。综述是给读者建立心智模型的；教学单元是给读者动手的。两者目标不同。

## 其他层次

- **`docs/reference/ragflow-notes/`** — 工业级 RAGFlow 源码摘录（按章对应）。不在默认阅读路径上，想做深读时翻。

## 当前内容

- **`01_what_is_rag.md`** — RAG 技术综述：什么是 RAG、为什么、与 fine-tuning/长上下文如何权衡、技术演进的三个阶段、何时用 RAG / 何时不用。
