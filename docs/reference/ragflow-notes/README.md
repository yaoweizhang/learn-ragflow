# RAGFlow 源码摘录

## 用途

本目录**抄录 [RAGFlow](https://github.com/infiniflow/ragflow) 项目中的关键源码片段**，用于：

1. 给每章 MVP 一个"工业级对照"——为什么 RAGFlow 这么写、它解决了什么工程问题；
2. 给想深入 RAG 实现的读者提供一份带注释的阅读路径。

> **不是教程主线**。每章的 `<topic>.py` 完全独立可跑；本目录是阅读材料，不是依赖。

## 引用规范

每个摘录文件遵循：

```markdown
# <主题>

## 一句话
（一句话讲清这文件的核心点 + 跟 MVP 的对比）

## 来源
- 仓库：https://github.com/infiniflow/ragflow
- 模块：`<module-path>`（只标模块路径，不标行号）
- 关联：与本仓库 `<对应章节>/<文件>.py` 对照阅读

## 代码
```python
<抄录的源码 + 注释>
```

## 为什么这样写
- 关键设计点
- 跟本章 MVP 的差异
```

**约定**：
- **不写代码行号**。RAGFlow 持续重构，行号会过时；保留模块路径 + 函数 / 类名作为稳定锚点。
- 摘录保留**原始缩进和注释**，不要"美化"。
- 每段摘录**聚焦一个主题**（一个函数、一个类、一段算法），不抄大段。

## 阅读顺序

按 s02 → s03 → ... → s12 顺序读最顺；每个文件首部"一句话"会点出它对应仓库哪一章。

## 免责声明

RAGFlow 仍在活跃演进。本目录的摘录**可能随时间过时**——函数签名、内部 API、模块路径都会变。
引用时请到原仓库对照验证（特别是间隔 6 个月以上的引用）。如发现内容已大幅漂移，欢迎提 PR 刷新。
