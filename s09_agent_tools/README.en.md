# s09 Agent & Tool Use

After s08, the RAG pipeline is fully wired: embed → retrieve → rerank → prompt → answer. But not every question needs the document store — greetings, simple math, and chit-chat go straight through to the LLM with no retrieval. This chapter lets the LLM itself decide whether to call a `retrieve` tool or just `finish`. The `run_agent` function implements a classic ReAct loop: every turn the model emits a `Thought` / `Action` / `ActionInput` triple, the loop parses the action, routes it to the right tool (the retrieve tool is s05-s07 composed), and feeds the observation back into the message history. The loop is bounded by `max_steps=5`. The chapter README walks through the demo queries (one that needs retrieval, one that should skip it), the three real-world failure modes (unstable LLM output parsing, tool explosion, multi-step recovery), and how RAGFlow's pluggable DAG-based agent design — using OpenAI-compatible `tool_calls` instead of string parsing — addresses them; details in [ragflow_notes/agent_tools.md](../ragflow_notes/agent_tools.md).
