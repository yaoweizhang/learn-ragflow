# s12 Deployment — Docker Compose packaging the MVP as a service

Chapter 12 wraps the s08 `answer()` function in a FastAPI app (`s12_deployment/app.py`, exposing `POST /qa`) and packages the whole tutorial project (s01–s11 source, requirements.txt, samples, pre-built `_chroma` index) into a single Docker image via `Dockerfile` + `docker-compose.yml`. `code.py` is a friendly launcher that pre-checks `.env` and `_chroma/` and runs `docker compose up --build`; once the container is up, anyone with `curl` can hit the service without installing Python, downloading models, or building the index locally. The trade-off versus RAGFlow's 10-service deployment (ES / Infinity / MySQL / Redis / MinIO / executor / etc.) is single-container simplicity — no multi-tenancy, no queue, no object store, no sandboxed tools — and the README's "troubleshooting" + "real-world problems" sections enumerate exactly what you'd add to make this production-ready (image layering, index prewarming, log aggregation, `/healthz`, auth, rate limiting). See `docs/reference/ragflow-notes/deployment.md` for the full RAGFlow-vs-MVP service comparison and `thinking_answers.md` for the production-readiness checklist.
