# samples/ — 共享样本文件

本教程**跨章复用**两个样本文件，用于保证每章的实验可复现、可对比。

## 为什么由读者自行准备？

为避免分发可能受版权保护的资料，**这两个文件不随仓库一起提供**。请读者自行准备两个相似规格的文档，命名为下面这两个文件名（**严格小写、下划线分隔，不要改动**）。

## 期望规格

### 1. `server_whitepaper.pdf`

- **格式**：PDF
- **语言**：中文
- **页数**：3–5 页
- **必须包含**：至少 1 个表格
- **主题建议**：任意产品白皮书；教程默认使用"紫光恒越 R3630 G5 服务器白皮书"

### 2. `disclosure.docx`

- **格式**：DOCX（Word 文档）
- **语言**：中文
- **内容**：财务披露文档
- **必须包含**：多个段落 + 多个表格（用于测试结构感知分块、表格解析等能力）

## 使用约定

- **每章都会读取这两个文件**。文件名是硬编码的约定，请不要改动。
- 不要把样本提交到 git（已在 `.gitignore` 中通过 `samples/_derived/` 等规则屏蔽部分派生产物；样本本身请放在本地）。
- 如果你的样本字段抽取不理想，可以替换内容，但**保持文件名不变**。

## 验证

把文件放到 `samples/` 后，运行：

```bash
ls -lh samples/
# 应该看到：
# server_whitepaper.pdf
# disclosure.docx
```

随后进入任意章节目录运行 `code.py` 即可开始实验。
