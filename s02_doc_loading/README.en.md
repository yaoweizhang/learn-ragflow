# s02 — Document Loading (PDF + DOCX)

Loads PDF and DOCX into structured dicts (each carrying `text`, `page`, `source` keys) using the lightweight `pypdf` and `python-docx` libraries. The naive reader approach is enough for clean, born-digital PDFs and simple DOCX files, but has well-known limits: no OCR (scanned PDFs return empty strings), lossy tables (PDF table cells reorder by drawing order; DOCX tables live in a separate `tables` collection, not `paragraphs`), and unfiltered headers/footers that pollute every chunk. RAGFlow addresses all three in `deepdoc/parser/pdf_parser.py` with a multi-backend strategy (`PlainParser`, `VisionParser`, `TxtParser`) that renders pages to images and runs OCR + layout + table-structure recognition when the text layer is empty — covered in `../ragflow_notes/deepdoc_pdf_parsing.md` and foreshadowing s11 on multimodal ingestion.
