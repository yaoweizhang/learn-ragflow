# s11 Multimodal — Table extraction (pdfplumber) + OCR (tesseract)

s02's `pypdf` flattens tables into a single string of text (losing the row/column structure), and scanned PDFs have no text layer at all — `pypdf.extract_text()` returns the empty string. These two "multimodal" inputs — structured tables and image-baked text — are where the classic RAG pipeline breaks down most often. `s11_multimodal/code.py` provides two independent helpers: `extract_tables(pdf_path)` uses `pdfplumber.extract_tables()` to pull per-page row/column data into `list[{"page", "rows"}]`, and `ocr_image(image_path)` shells out to `pytesseract.image_to_string(..., lang="chi_sim+eng")` for scanned/image content. The OCR half needs the system `tesseract` binary on PATH (Windows installer from UB-Mannheim; macOS `brew install tesseract tesseract-lang`; Linux `apt install tesseract-ocr tesseract-ocr-chi-sim`); the table half just needs `pdfplumber`. See the zh README for troubleshooting and the ragflow_notes link for how RAGFlow handles tables (dedicated TableStructureRecognizer + per-cell OCR) and OCR fallback (detect garbled text and route to vision backends).
